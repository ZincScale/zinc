# Zinc — Deployment Guide

## Overview

Zinc transpiles `.zn` files to `.java`, compiles with javac, and runs on Java 25. For production, Quarkus + GraalVM native-image produces single-binary deployments.

```
.zn source → zinc build → .java → javac → .class → run / package
```

---

## Quick Reference

| Command | Output | Use case |
|---|---|---|
| `zinc run script.zn` | Runs immediately | Development, scripts |
| `zinc build script.zn` | Compiled .class files | Library, integration |
| `zinc build --native` | GraalVM native binary | Production, containers |
| `zinc build --docker` | Dockerfile + JAR | Container deployments |
| `zinc build --k8s` | Dockerfile + K8s manifest | Kubernetes deployments |

---

## 1. Run Directly

The simplest deployment — just run the `.zn` file:

```bash
zinc run script.zn
zinc run script.zn -- arg1 arg2
```

**What happens:**
1. Zinc transpiles `script.zn` → `.java` files
2. Compiles with `javac`
3. Runs with `java` (Java 25, virtual threads enabled)

**When to use:** Development, cron jobs, one-off scripts, CI/CD pipelines.

---

## 2. JAR — JVM Distribution

Build a self-contained JAR for deployment on any machine with Java 25:

```bash
zinc build myproject/
# produces myproject.jar

java -jar myproject.jar
```

**When to use:** Services running on JVM infrastructure, when startup time isn't critical.

---

## 3. GraalVM Native Image — Single Binary

Compile to a native binary via Quarkus + GraalVM. Fast startup, low memory:

```bash
zinc build myproject/ --native
# produces myproject (native binary)

./myproject
```

**What happens:**
1. Zinc transpiles all `.zn` → `.java`
2. Mill builds the project
3. GraalVM `native-image` compiles to a standalone binary

**Fallback:** If native-image fails (reflection-heavy libraries), JLink produces a smaller JVM bundle:

```bash
zinc build myproject/ --jlink
# produces myproject/ directory with minimal JVM + app
```

| | JAR | Native Image | JLink |
|---|---|---|---|
| Startup | ~500ms | ~10ms | ~200ms |
| Memory | JVM overhead | Minimal | Reduced JVM |
| Binary size | Small + JVM | ~20-50 MB | ~40-80 MB |
| Compatibility | All libraries | Some restrictions | All libraries |

**When to use:** Production microservices, CLI tools, Lambda/serverless functions.

## 4. Docker — Container Image

```bash
zinc build myproject/ --docker

docker build -t myapp .
docker run myapp
```

**What `zinc build --docker` generates:**

`Dockerfile`:
```dockerfile
FROM eclipse-temurin:25-jre-alpine

WORKDIR /app
COPY target/myapp.jar .

EXPOSE 8080
CMD ["java", "-jar", "myapp.jar"]
```

For native image:
```dockerfile
FROM ghcr.io/graalvm/native-image:25 AS build
WORKDIR /app
COPY . .
RUN native-image -jar myapp.jar myapp

FROM gcr.io/distroless/base
COPY --from=build /app/myapp /myapp
CMD ["/myapp"]
```

**When to use:** Microservices, cloud deployments, CI/CD containers.

---

## 5. Kubernetes — Full Deployment

```bash
zinc build myproject/ --k8s

# Build and push
docker build -t myregistry/myapp:latest .
docker push myregistry/myapp:latest

# Deploy
kubectl apply -f myapp-deployment.yaml
kubectl get pods -l app=myapp
```

**What `zinc build --k8s` generates:**

`Dockerfile` (same as above)

`myapp-deployment.yaml`:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
  labels:
    app: myapp
spec:
  replicas: 1
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myapp:latest
        resources:
          requests:
            memory: "64Mi"
            cpu: "100m"
          limits:
            memory: "256Mi"
            cpu: "500m"
```

**When to use:** Production services, data pipelines, scheduled jobs in K8s.

---

## 6. Shebang — Executable Scripts

Make `.zn` files directly executable on Unix:

```zinc
#!/usr/bin/env zinc run
print("Hello from zinc!")
```

```bash
chmod +x script.zn
./script.zn
```

**When to use:** Utility scripts, automation, replacing bash scripts.

---

## Project Structure

A typical Zinc project:

```
myapp/
├── main.zn              # entry point
├── models.zn            # data classes
├── utils.zn             # utility functions
├── zinc.toml            # project config (Mill build)
├── Dockerfile           # generated by zinc build --docker
└── myapp-deployment.yaml # generated by zinc build --k8s
```

## CI/CD Example

GitHub Actions workflow:

```yaml
name: Build and Deploy
on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4

    - name: Install Zinc
      run: go install github.com/victorybhg/zinc/cmd/zinc@latest

    - name: Set up Java 25
      uses: actions/setup-java@v4
      with:
        distribution: temurin
        java-version: 25

    - name: Build and type check
      run: zinc build myproject/

    - name: Build Docker image
      run: |
        zinc build myproject/ --docker
        docker build -t myapp:${{ github.sha }} .

    - name: Push to registry
      run: docker push myregistry/myapp:${{ github.sha }}

    - name: Deploy to K8s
      run: kubectl set image deployment/myapp myapp=myregistry/myapp:${{ github.sha }}
```
